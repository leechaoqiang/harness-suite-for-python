# 变更摘要准备

## 用途
分析代码变更，生成结构化评审摘要，为后续专项评审提供上下文。

## 输入

- OpenSpec change-id
- 代码变更范围

## 执行流程

### 第一步：收集变更信息

```
1. 读取 proposal.md / design.md / tasks.md
2. 获取 git diff（如有 git）
3. 或根据 tasks.md 推断变更范围
```

### 第二步：分析变更

```
分类变更：
  ├─ 路由/API 层变更
  ├─ Service 层变更
  ├─ Repository/Model 层变更
  ├─ Schema/类型定义变更
  ├─ 数据库迁移变更
  ├─ 配置变更
  └─ 测试变更
```

### 第三步：输出摘要

```markdown
# 变更摘要：<change-id>

## 变更概览
- 变更类型：[新增 / 修改 / 重构 / 修复]
- 影响模块：
- 变更文件数：

## 分层影响

### API 层
- 变更文件：
- 新增/修改接口：

### Service 层
- 变更文件：
- 新增/修改逻辑：

### 数据层
- 变更文件：
- 数据库变更：

### 类型定义
- 新增/修改 Schema：

## 测试覆盖
- 新增测试：
- 修改测试：
- 未覆盖变更：

## 高风险标记
- [ ] 涉及数据库迁移
- [ ] 涉及认证/鉴权
- [ ] 涉及批量操作
- [ ] 涉及外部 API 调用
- [ ] 涉及异步任务
```
